Spring Boot - Dockerizar con base de datos
******************************************
(Docker y docker-compose)

Para esto se tiene un app hecho en Spring Boot llamado notas-seguras con conexión a la BBDD MySQL notas-seguras.

Configuración del Docker
========================

Verificar que en Usuarios/<Tu usuario>/AppData/Roaming/Docker/settings.json las propiedades estén colocadas de esta forma:

"integratedWslDistros" : [ ]
"enableIntegrationWithDefaultWslDistro" : false,

Asimismo, en /Usuarios/<Tu usuario>/.docker/config.json debe estar:

{
  "credStore": "desktop"
}

"credStore" en lugar de "credsStore" (en caso aplique).


Definir los archivos de Docker
==============================

Dockerfile
----------

FROM maven:3.8.3-openjdk-17

WORKDIR /notas-seguras
COPY . .
RUN mvn clean install -DskipTests

CMD mvn spring-boot:run

Lo que contiene:

- FROM es el compilador de Java empleado para la generación del build.
- WORKDIR es el directorio de trabajo, en este caso es notas-seguras (raíz del proyecto).
- COPY genera una copia de todos los elementos a la imagen creada.
- RUN y CMD son ejecuciones de comandos requeridos para el build/install y la ejecución del servicio en el contenedor.

Este archivo contiene la imagen que permitirá generar el contenedor que permitirá la disponibilidad de una aplicación Spring Boot.

docker-compose.yml
------------------

version: '4.22'

services:
  mysqldb:
    image: mysql:5.7
    restart: unless-stopped
    env_file: ./.env
    environment:
      - MYSQL_ALLOW_EMPTY_PASSWORD=1
      - MYSQL_ROOT_PASSWORD=$MYSQLDB_ROOT_PASSWORD
      - MYSQL_DATABASE=$MYSQLDB_DATABASE
    ports:
      - $MYSQLDB_LOCAL_PORT:$MYSQLDB_DOCKER_PORT
    volumes:
      - db:/var/lib/mysql
  app:
    depends_on:
      - mysqldb
    build: .
    restart: on-failure
    env_file: ./.env
    ports:
      - $SPRING_LOCAL_PORT:$SPRING_DOCKER_PORT
    environment:
      SPRING_APPLICATION_JSON: '{
        "spring.datasource.url"  : "jdbc:mysql://mysqldb:$MYSQLDB_DOCKER_PORT/$MYSQLDB_DATABASE?useSSL=false",
        "spring.datasource.username" : "$MYSQLDB_USER",
        "spring.datasource.password" : "$MYSQLDB_ROOT_PASSWORD",
        "spring.jpa.hibernate.ddl-auto" : "create-drop"
      }'
    volumes:
      - .m2:/root/.m2
    stdin_open: true
    tty: true

volumes:
  db:

- MYSQL_ALLOW_EMPTY_PASSWORD=1 significa que puede usarse una clave vacía
- notar que las variables con "$" provienen de un archivo .env en la raíz del proyecto.

.env
----

MYSQLDB_USER=root
MYSQLDB_ROOT_PASSWORD=
MYSQLDB_DATABASE=notas-seguras
MYSQLDB_LOCAL_PORT=3306
MYSQLDB_DOCKER_PORT=3306

SPRING_LOCAL_PORT=4002
SPRING_DOCKER_PORT=4002

Gestión de las imágenes y contenedores
======================================

Crear imágenes y contenedor
---------------------------
docker-compose up

Ejecución en background:
docker-compose up -d

Monitorear ejecuciones
----------------------
Contenedores activos:
docker ps

Borrar contenedores
-------------------
Contenedor específico:
docker kill :container_id

Todos los contenedores:
docker-compose down

Ver todas las imágenes
----------------------
docker images

Ingresar al terminal de un contenedor particular
------------------------------------------------
docker exec -it :container_id bash

Logs
----
docker-compose logs

Detener contenedor
------------------
docker-compose stop

Iniciar contenedor
------------------
docker-compose start

Reniciar contenedor
------------------
docker-compose restart
